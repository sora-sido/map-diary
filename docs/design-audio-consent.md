# 音声書き起こし・会話記録の同意/削除ポリシー(Step9以降の実装前提整理)

会話の録音・書き起こしは相手の同意なく行うとプライバシー・法的リスクが高い機能。
実装はStep9以降とし、着手前に必ず以下のUXとポリシーを固めてから進める。

## 録音同意のUI/UXフロー案
1. MTG開始前に「録音を開始しますか?」の確認ダイアログを表示し、
   同意した場合のみ録音を開始する(デフォルトはOFF)。
2. 録音中は画面上に常時「録音中」の視覚的インジケーター(赤い点滅バッジ等)を表示し、
   本人がいつでも即座に停止できるボタンを常設する。
3. 対面の相手にも録音していることが伝わるよう、
   スマホ画面を見せる/口頭で伝える運用を前提とし、アプリ側は
   「相手に伝えましたか?」のチェックを記録開始前に求める
   (`meeting_transcripts.consentGiven`に保存)。
4. 同意が得られなかった/不明な場合は録音データを保存せず、
   手動メモのみを残せるようにする。

## 保存期間・削除ポリシー
- 生の音声データは書き起こし完了後、原則**即時削除**(書き起こしテキストのみ保持)。
- 書き起こしテキスト(`meeting_transcripts.rawText`)は
  `retentionExpiresAt`をデフォルト90日後に設定し、期限が来たら
  日次バッチで自動削除する。
- ユーザーがいつでも個別の書き起こしを即時削除できるUIを用意する。
- 削除操作は物理削除(論理削除ではない)とし、バックアップにも
  保持期限を連動させる。
