# OpenAI API コスト管理方針(Step7以降の実装前提整理)

個人利用アプリのためコストは小さいはずだが、無制限の呼び出しを避けるため
以下を設計方針とする。実装はStep7(AI要約)着手時に行う。

## 呼び出し頻度
- **1日1回、夜間バッチのみ**(例: 深夜2時)でその日の要約を生成する。
  ユーザー操作のたびにAPIを呼ぶ設計にはしない。
- 手動での「今すぐ要約し直す」再生成ボタンは許可するが、
  同一`daily_logs.date`に対して1時間あたり最大1回などのレート制限を設ける。

## 1日あたりの想定トークン数
- 入力: その日のカレンダー予定(数件〜十数件)・日記本文・会話要約の元テキストを
  結合しても概ね2,000〜4,000トークン程度と想定。
- 出力: 要約文+キーワード+感情分析で500〜1,000トークン程度。
- 1日あたり合計 概ね3,000〜5,000トークン。個人利用(1ユーザー/日1回)なので
  月間でも15万トークン程度に収まる見込み。

## キャッシュ方針
- 生成結果は`ai_summaries`テーブルに`dailyLogId`単位で保存し、
  同じ日の再表示ではAPIを再呼び出しせずDBの値を返す。
- 入力データ(日記・カレンダー等)に変更がない限り再生成しない。
  変更検知は「最後にAI要約を生成した時刻以降に該当日の
  diary_entries/calendar_events/meetingsが更新されたか」で判定する。
- モデル・トークン使用量は`ai_summaries.model`/`tokenUsage`に記録し、
  将来コストが想定を超えていないか確認できるようにする。

## モデル選定
- コスト最優先のため、要約・キーワード抽出・感情分析には
  軽量モデル(例: GPT-4o mini相当)を第一候補とし、
  必要に応じて上位モデルへの切り替えを環境変数で選べるようにする。
